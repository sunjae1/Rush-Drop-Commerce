# 잠재 오류 감사 보고서

## 1. 현재 사용자 확인용 전용 API가 아직 없음

심각도: 높음

현재 세션 확인은 `/api` 응답에 의존합니다.  
이 엔드포인트는 사용자 정보와 상품 목록을 함께 보내고 있어서, 순수한 인증 확인 용도로는 무겁습니다.

대응:

- 다음 단계에서 `/api/me` 또는 `/api/session` 같은 전용 엔드포인트 검토
- 이번 작업에서는 제안만 남기고 구현은 보류

## 2. 카테고리 필터를 진짜 상품 필터처럼 만들 수 없음

심각도: 중간

현재 확인한 `ItemDto`에는 카테고리 정보가 없습니다.  
그래서 카테고리 API는 받더라도 상품을 카테고리별로 정확히 필터링할 수 없습니다.

대응:

- 백엔드에서 상품 DTO에 카테고리 연결 정보 추가
- 또는 상품 조회 API에 카테고리 기반 필터 파라미터 추가

## 3. 프런트 단에서 작성자 판별이 이름 기반임

심각도: 중간

게시글/댓글 수정 버튼 노출 여부는 현재 `user.name`과 작성자 이름 문자열 비교에 의존합니다.  
이름이 같은 사용자가 있으면 UI 표시가 부정확할 수 있습니다.

대응:

- 백엔드 DTO에 작성자 ID 포함
- 프런트는 ID 기준으로 본인 여부 판단

## 4. 별도 도메인 배포 시 세션 인증이 바로 안 맞을 수 있음

심각도: 중간

개발 환경은 Vite 프록시로 문제를 피하지만, 프런트와 백엔드를 다른 도메인으로 분리 배포하면 CORS, 쿠키, SameSite 설정 이슈가 생길 수 있습니다.

대응:

- 같은 도메인 리버스 프록시 운영
- 또는 백엔드에 CORS 및 쿠키 정책 추가 설정

## 5. React Router 테스트 경고가 보임

심각도: 낮음

테스트는 통과하지만 React Router v7 future flag 관련 경고가 출력됩니다.

대응:

- 추후 React Router 업그레이드 시 future flag 검토
- 현재는 실패가 아니라 경고라서 작업 중단 사유는 아님

## 6. `npm audit` 기준 중간 수준 취약점 5건

심각도: 낮음

의존성 설치 시 `npm audit` 결과 moderate 취약점 5건이 표시되었습니다.

대응:

- 배포 전에 `npm audit` 상세 확인
- 영향 패키지와 실제 런타임 노출 여부 검토
